1.1 Introduction to R

Introduction to R

  • Todays data analysts need to be able to:

    • Access data from a wide range of sources
    • Merge the pieces of data together
    • Clean and annotate them
    • Analyze them with the latest methods

Introduction to R

What is R?

  • R is a free software environment for statistical computing and graphics.

  • R has become the world-wide language for statistics, predictive analytics, and data visualization.

What can you do with R?

  • Do calculations

  • Perform statistic analysis

  • Create graphics

  • Write your own functions

Why R?

  • Technical advantages: free, open-source, available for all OSs

  • Efficient functions and data structures for data analysis

  • Powerful graphics

  • Access to fast growing number of analysis packages

Comparison to other software

  • Stata: It can handle all kinds of econometrics research. Many packages available

  • SAS: It is really good for big data set, but the grammar is really weird.

  • Matlab: I could not compare it directly to R since it is mainly a mathematical and engineering software instead of a statistic one. The only problem may be the size.

  • Others do not be very popular in economic field. SPSS may be good for management student; Eviews specializes more in time series analysis.

  • And the most important factor: you need to pay for them.

1.3 Working with R

Working with R

  • There is an important difference in philosophy between R and the most other econometrics packages.

  • With many packages, an analysis will lead to a large amount of output containing info on estimation, model diagnostics, specification tests, etc.

  • In R, an analysis is normally broken down into a series of steps. Intermediate results are stored in objects, with minimal output at each step.

  • Instead, the objects are further manipulated to obtain the info required.

Working with R

  • R is a case-sensitive, interpreted language.

  • Most functionality is provided through built-in and user-created functions.

  • R uses both <- and = for assignments. = is recommended

  • Comments are preceded by the # symbol.

1.4 Packages

Packages

  • Packages are collections of R functions, data, and compiled code in a well-defined format.

    install.packages()
library()

Packages

Common mistakes

  • Using the wrong case: help(), Help(), HELP()

  • Forgetting to use quote marks when they are needed: install.packages("stringr")

  • Forgetting to include the parentheses in a function call: help(), help

  • Using the \ in a pathname on windows:
    • setwd("c:\myprogram") #does not work
    • setwd("c:/myprogram"), or setwd("c:\\myprogram")

  • Using a function from a package that is not loaded: library()
