2.1 Understanding datasets

Understanding datasets

  • A dataset is usually a rectangular array of data with rows representing observations and columns representing variables.

  • Well use the terms observations and variables to name the rows and columns.

  • R contains a wide variety of structures for holding data, including scalars, vectors, matrices, arrays, data frames, and lists.

  • The data types or modes that R can handle include numeric, character, logical, complex, and raw.

2.2 Data structures

Data structures

Vectors

  • Vectors are one dimensional arrays that can hold numeric data, character data, or logical data.

  • The combine function c() is used to form the vector.

    • a = c(1, 2, 5)
    • b = c("one", "two", "five")
    • c = c(TRUE, FALSE, TRUE)

  • The data in a vector must only be one type or mode, and you can not mix modes in the same vector.

  • Scalars are one-element vector.

Vectors

  • You can refer to elements of a vector using a numeric vector of positions within brackets.
a <- c(1, 2, 5, 3, 6, -2, 4)
a[3]
a[c(1, 3, 5)]
a[2:6]

Data frames

  • A data frame is more general than a matrix in that different columns can contain different modes of data.
patientID <- c(1, 2, 3, 4)
age <- c(25, 34, 28, 52)
diabetes <- c("Type1", "Type2", "Type1", "Type1")
status <- c("Poor", "Improved", "Excellent", "Poor")
patientdata <- data.frame(patientID, age, diabetes, status, row.names = patientID)
patientdata

  • It can get tired to type patientdata$ at the beginning of every variable name, so shortcuts are available.

  • The attach() function adds the data frame to the R search path:

  • This could also be written as:

  • The detach() function removes the data frame from the search path.

Data frames

  • Of course, here we use the base graph function plot() in the example. If we use qplot() from ggplot2 package, we can set data=mtcars and do not need the attach() and detach() functions.

Factors

  • As you have seen, variables can be described as nominal, ordinal, or continuous.

    • Nominal variables are categorical, without an implied order.
    • Ordinal variables imply order but not amount.
    • Continuous variables can take on any value within some range, and both order and amount are implied.

  • Categorical and ordered categorical variables in R are called factors.

Factors

  • The function factor() stores the categorical values as a vector of integers, and an internal vector of character strings mapped to these integers.
diabetes <- c("Type1", "Type2", "Type1", "Type1")
diabetes <- factor(diabetes)
diabetes
## [1] Type1 Type2 Type1 Type1
## Levels: Type1 Type2
status <- c("Poor", "Improved", "Excellent", "Poor")
status <- factor(status, ordered = TRUE)
status
## [1] Poor      Improved  Excellent Poor     
## Levels: Excellent < Improved < Poor

Factors

  • By default, factors levels for character vectors are created in alphabetical order.

  • You can override the default by specifying a levels option.

status <- factor(status, ordered = TRUE, levels = c("Poor", "Improved", "Excellent"))
status
## [1] Poor      Improved  Excellent Poor     
## Levels: Poor < Improved < Excellent

Lists

  • A list is an ordered collection of objects to allows you to gather a variety of them under one name.
g <- "My First List."
h <- c(25, 26, 18, 39)
j <- matrix(1:10, nrow = 5)
k <- c("one", "two", "three")
mylist <- list(title = g, ages = h, j, k)

  • A note for programmers

    • The (.) has no special significance in object name, but the ($) has a analogous meaning.
    • R does not provide multiline or block comments.
    • Assigning a value to a nonexistent element of a data structure, will expand it to accommodate the new value.
    • R does not have scalar values, but one element vectors.
    • Indices in R start at 1, not at 0.
    • Variables can not be declared.

2.3 Data input

Data input

Data input

  • The best way to read other files is to export it to a comma-delimited file and import it to R.

  • You can import data from delimited text files using read.table()

    grades <- read.table("studentgrades.csv", header = TRUE, sep = ",", row.names = "studentID", 
    stringsAsFactors = FALSE)

  • See help(read.table) for details.

  • The foreign package can help you too.
